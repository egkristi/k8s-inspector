# Security Analyzer

The Security Analyzer continuously monitors your Kubernetes clusters for misconfigurations and vulnerabilities. It evaluates the cluster against industry standards such as the CIS Kubernetes Benchmark and NSA/CISA Hardening Guidelines.

## Core Capabilities

### 1. Pod Security Standards (PSS) Enforcement
The analyzer detects pods and workloads running with elevated privileges.
- **Privileged Containers:** Detects `securityContext.privileged: true`.
- **Root User Execution:** Detects containers running as `root` instead of `runAsNonRoot: true`.
- **Host IPC/Network/PID:** Alerts on containers sharing the host namespaces.

### 2. Network Isolation
In Kubernetes, pods can communicate with each other freely by default. The Security Analyzer ensures your cluster follows the principle of least privilege for network traffic.
- **Default Deny-All:** Verifies that every namespace contains a baseline NetworkPolicy denying unexpected traffic.
- **Unprotected Endpoints:** Highlights exposed services lacking policy restrictions.

### 3. RBAC Auditing
Role-Based Access Control misconfigurations are a common attack vector.
- **Wildcard Permissions:** Detects `ClusterRole` or `Role` bindings that use `"*"` for verbs or resources (e.g., full cluster admin access).
- **Privilege Escalation:** Highlights roles that grant the ability to create other roles or modify cluster role bindings.

## How it works

The analyzer is a stateless Python module (`backend/app/analyzers/security_analyzer.py`) that executes via FastAPI background tasks.

1. **Telemetry Collection:** It fetches definitions of running Pods, Namespaces, NetworkPolicies, and Roles from the target cluster.
2. **Evaluation Engine:** It processes these objects against a set of predefined rules matching the CIS benchmarks.
3. **Scoring:** The cluster is given a health score out of 100. Deductions are calculated based on severity:
   - *Critical:* -10 points (e.g., Privileged containers, Wildcard RBAC)
   - *High:* -5 points (e.g., Root execution, Missing NetworkPolicies)
   - *Medium/Low:* -2/-1 points.

## Automated Remediation

Many security insights generated by this analyzer can be fixed automatically using Playbooks. For example, if a namespace lacks a default-deny NetworkPolicy, the `apply-default-deny-netpol` playbook can be triggered to automatically inject it.
